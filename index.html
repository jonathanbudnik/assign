    The main goal of The Web is simplify sharing research results over the internet. To accomplish that, Tim Berners-Lee (the web inventor) needed to determine how the documents were going to be represented, where were they going to be located and how they were going to communicate. For each of them he created a different protocol.

    Document representation: HTML.

    It is a way to organize the different elements of a Website through the implementation of different HTML Element, each of them represented by different marks. Each mark can display images, links or simply text, what helps the document keep organize. The HTML documents follow a Tree order, primary reason why it is so easy to read them and analyze them.

    Location of the documents: URL.

    Is a reference (an address) to a resource on the Internet.

    Communication: HTTP.

    Normally implemented over a TCP connection. The typical browser-server interaction is:

    1. User enters Web address in browser
    2. Browser uses DNS to locate IP address
    3. Browser opens TCP connection to server
    4. Browser sends HTTP request over connection
    5. Server sends HTTP response to browser over connection
    6. Browser displays body of response in the browser

  • Question 7

    There are currently two version of Internet Protocol (IP): IPv4 and a new version called IPv6. IPv6 is an evolutionary upgrade to the Internet Protocol. IPv6 will coexist with the older IPv4 for some time.

    IPv4 is the most widely deployed Internet protocol used to connect devices to the Internet. IPv4 uses a 32-bit address scheme allowing for a total of 2^32 addresses (just over 4 billion addresses). With the growth of the Internet it is expected that the number of unused IPv4 addresses will eventually run out. That's why a new Internet addressing system Internet Protocol version is being deployed to fulfill the need for more Internet addresses: The IPv6.

    IPv6 addresses are 128-bit IP address written in hexadecimal and separated by colons. An example IPv6 address could be written like this: 3ffe:1900:4545:3:200:f8ff:fe21:67cf. Other technological improvements are showing below:

    • No more NAT
    • Auto-configuration
    • No more private address collisions
    • Better multicast routing
    • Simpler header format
    • Simplified, more efficient routing
    • True quality of service (QoS)
    • Built-in authentication and privacy support
    • Flexible options and extensions
    • Easier administration
